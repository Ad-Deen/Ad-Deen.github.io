Project ORCA.

Autonomous Underwater Vehicle System: Engineered for prolonged operation, seabed mapping, and robust marine exploration.

01 // SYSTEM_ARCHITECTURE

Hardware & Control Strategy

ORCA is an in-house Autonomous Underwater Vehicle (AUV) designed as a neutral-buoyant platform. A custom buoyancy control box and a variable center-of-gravity (COG) mechanism allow for fine adjustments to maintain stable underwater operation while securely housing thrusters and sensors.

The control architecture is bifurcated: a Jetson Nano handles high-level processing (visual perception, sensor fusion, decision-making), while a dedicated ESP32 module executes low-level thruster commands and relays real-time sensor feedback.

Currently, to accommodate testing constraints in turbid environments, the system operates as a fully functional tethered ROV controlled via Ethernet. This setup successfully validates sensor readings, thruster responsiveness, and telemetry, proving the platform's capability to support future untethered autonomy.

02 // DEVELOPMENT_MILESTONES

A

Mechanical Stack

PETG Frame Variable COG Neutral Buoyancy 6-DOF Manipulator

The mechanical design evolved through iterative improvements focusing on buoyancy, stability, and maneuverability. Replacing heavy aluminum frames, Version 2 utilizes an optimized 3D-printed PETG frame.

Integrated a variable center-of-gravity system and modular electronics chambers to achieve near-neutral buoyancy. A custom 6-DOF manipulator was also developed to enable precise underwater physical interactions.

B

Electrical Stack

Custom PCB Power Isolation Dual ESP32 Li-ion 4S

Designed for robust underwater operation, the electrical system integrates two 4S Li-ion batteries with modular buck converters to supply power across ESCs, ESP32 controllers, and the Jetson Nano.

A custom PCB handles high-current power distribution and relay isolation. Dual ESP32s manage thruster control and manipulator actuation, communicating with the Jetson Nano via UART for real-time edge processing.

C

Simulation Environment

Gazebo Isaac Sim Blender Hydrodynamics

Developed a custom underwater simulation environment spanning Gazebo and NVIDIA Isaac Sim. The seabed topology was modeled in Blender, with physics and hydrodynamics applied in Gazebo to replicate water resistance and buoyancy.

This digital twin allowed for the integration and testing of simulated visual feedback, validating the ROS control architecture and navigation stack prior to physical pool deployment.

D

ROS Software & Navigation Stack

ROS 2 Zero-Copy Memory Potential Fields GPU Accel

Engineered a real-time navigation stack optimized for edge-GPU processing. Sensor data is efficiently streamed to the GPU utilizing DMA and zero-copy memory architecture for parallel computation.

Trajectory generation is driven by a potential fields method—balancing goal attraction and obstacle repulsion. Hard kinematic and dynamic constraints are applied continuously to ensure planned trajectories remain feasible for the AUV’s physical limits.
