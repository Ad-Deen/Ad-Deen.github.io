Autonomous Swarm Robotic Drone System

Fly together – Perceive faster – Report quicker – Reconstruct better – Agile tracking

Technical Highlights

Our swarm drone system enables autonomous 3D mapping and navigation using stereo vision. Each drone captures stereo frames that are processed on a ground PC to generate depth, extract 2D/3D features, and estimate odometry using frame-to-frame matching and PnP. The navigation stack uses this odometry and the evolving map to issue short-term commands (e.g., “fly up 30 cm”), allowing the drone to build the environment model while navigating in real time. This pipeline enables robust, self-updating 3D world reconstruction and short-horizon autonomy, suitable for search, rescue, and large-area inspection tasks.

Snapshots of the Progress

We have a step-by-step documentation of our project in the following sections. Feel free to dive deep into our documentation in the GitHub links however you like!

A : Custom Flight Controller Testing for waypoint based Navigation

I designed and implemented a lightweight custom flight controller for UAV stabilization and navigation. The system includes PID-based stabilization loops, teleoperation via ROS 2, and prototype autonomous control pipelines tested in Gazebo simulation. This allowed rapid experimentation with control strategies while keeping the design modular and open-source for further research. In the controller it takes a limits of way points and based on odometry it tries to reach the goal keeping balance and accuracy avoiding overshoot.

B : Monocular Depth Estimation Pipeline

To enable real-time 3D scene understanding during drone flights, I integrated the Depth Anything V2 model into a server-based pipeline. Video frames captured onboard the UAV are transmitted to the server, where dense depth maps are generated in real time. These depth estimations are used for scene reconstruction, odometry, and SLAM integration, allowing robust navigation in GPS-denied environments. The pipeline was designed to run efficiently on edge-GPU servers for seamless deployment in field operations.

C : Structure-from-Motion & Visual Odometry

This pipeline focuses on reconstructing 3D structures and estimating UAV motion using camera-only input. By leveraging feature extraction, triangulation, and bundle adjustment, the system builds consistent 3D maps from monocular image sequences. Visual Odometry is integrated to track drone position across frames, enabling localization without dependence on GPS. These techniques form the backbone of visual SLAM, helping UAVs operate in GPS-denied or cluttered environments with improved autonomy.

D : Custom Flight Controller Practical implementation

Due to limited funding, I developed a low-cost custom flight controller on an ESP32 module. I selected the ESP32 as the base platform since it has built-in Wi-Fi enabled communication system, eliminating the need for additional Visual/telemetry transmitter and receiver modules. This also helped minimize power consumption during flight. I integrated sensor interfaces for IMU, GPS, motor controllers, and camera, and built the firmware from scratch. The design was open-sourced on GitHub to support other researchers working on UAV projects under tight budgets.

Learn more

E : Custom Stereo vision setup with OV5640 camera

As part of Project Bee, we developed a custom-built stereo vision system using dual ESP32-S3-CAM modules with OV5640 sensors mounted on a custom PCB. The design enables synchronized frame capture, Wi-Fi based UDP transmission, and real-time reconstruction via ROS 2 nodes on a server PC. Missing frame chunks are patched using Gaussian interpolation to ensure stream continuity. This lightweight, low-cost stereo setup provides reliable depth perception for UAVs, feeding directly into 3D mapping, SLAM, and advanced perception pipelines.
