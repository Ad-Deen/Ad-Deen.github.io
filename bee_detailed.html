System Architectural Roadmap

Phase 1: Hardware & Physical Build (Completed)
We have successfully fabricated the physical UAV platform. This includes the integration of a global shutter stereo camera pair and the engineering of a custom made flight controller(no plug and play Px4/Ardupilot systems). The controller features a proprietary optical flow system built exclusively using ESP32-S3 and ESP-CAM modules keeping the unit price cheap.

Phase 2: VSLAM & Simulation (Current Focus)
Before deploying the full Visual-Inertial SLAM (VI-SLAM) stack onto the physical drones, the stereo perception pipeline is being rigorously tested and optimized within NVIDIA Isaac Sim. This sim-to-real validation ensures robust depth generation, odometry estimation , and environment mapping prior to live flight operations.

Phase 3: RaaS Ecosystem (Future Target)
The broader, generalized Robotics-as-a-Service (RaaS) infrastructure—encompassing multi-agent coordination, semantic world-modeling, and human-in-the-loop control interfaces—is slated for future development. My objective is to architect and scale this final ecosystem during my upcoming higher education research abroad.

>> PHASE 1: HARDWARE & PHYSICAL BUILD

A

ESP32 Flight Controller Hardware

ESP32 Firmware Sensor Fusion

Developed a low-cost custom flight controller from scratch on an ESP32 module. The ESP32's built-in Wi-Fi eliminates the need for separate telemetry modules, minimizing weight and power draw.

Integrated interfaces for IMU, GPS, motor controllers, and cameras. The design is open-sourced to support researchers working on hardware-constrained budgets.

B

Custom OV5640 Stereo Vision Module

ESP32-S3-CAM OV5640 Custom PCB UDP Wi-Fi

Developed a custom-built stereo vision system utilizing dual ESP32-S3-CAM modules with OV5640 sensors mounted on a custom PCB.

The design enables synchronized frame capture and Wi-Fi-based UDP transmission for real-time reconstruction via ROS 2 on a server. Missing frame chunks are patched using Gaussian interpolation to ensure stream continuity.

>> PHASE 2: VSLAM & SIMULATION

C

Custom Flight Controller Navigation

Gazebo / Isaac Sim PID Control ROS 2

Designed and implemented a lightweight custom flight controller for UAV stabilization and navigation. The system includes PID-based stabilization loops, teleoperation via ROS 2, and prototype autonomous control pipelines.

It takes limits of waypoints and, based on odometry, tries to reach the goal while maintaining balance and avoiding overshoot. This modular approach allows for rapid experimentation with control strategies prior to physical deployment.

🔗 View Documentation
D

Monocular Depth Estimation Pipeline

Depth Anything V2 Edge-GPU 3D Scene Understanding

To enable real-time 3D scene understanding during flight, I integrated the Depth Anything V2 model into a server-based pipeline. Video frames captured onboard the UAV are transmitted to the server, where dense depth maps are generated in real time.

These estimations feed directly into scene reconstruction, odometry, and SLAM integration, allowing robust navigation in GPS-denied environments.

E

Robust Visual-Inertial SLAM (VI-SLAM)

VI-SLAM IMU Fusion Decoupled Odometry Sparse Mapping

Engineered a custom Visual-Inertial SLAM pipeline, pivoting from standard PnP methods to a decoupled odometry technique fused directly with IMU telemetry.

This sensor fusion approach maintains high robustness even in low-feature environments. By drastically reducing keyframe triggers, the system heavily optimizes map storage memory while relying on a sparse 3D landmark architecture.

🔗 View VSLAM Stack
F

Real-Time Memory-Efficient Nav Map

SuperPoint / LightGlue O(1) Spatial Hashing Zero-Copy SHM OpenGL / Pangolin

Engineered a hardware-accelerated mapping engine and VIO pipeline for the Deploy & Forget (D&F) prototype. Running on a Jetson Orin Nano using Isaac Sim telemetry, the perception stack leverages SuperPoint, LightGlue, and a DinoV2 backbone to achieve a 3 cm tracking error threshold.

To process dense point clouds without bottlenecking the 60 Hz odometry loop, I built a decoupled voxel engine utilizing O(1) spatial hashing in C++ so RAM scales dynamically with physical volume. It uses Linux Shared Memory (/dev/shm) for zero-copy transport and custom OpenGL geometry shaders to drop CPU rendering overhead to near zero.

>> PHASE 3: ACTIVE INTEGRATION & RaaS TARGETS

G: Sim Integration & Autonomous Recharge

Active Development Isaac Sim Power Sim Obstacle Avoidance

Fusing the VIO Nav Map and flight stabilization within a dynamic Isaac Sim warehouse environment. Engineering a control loop that accepts user waypoints, actively computes simulated battery draw, executes dynamic obstacle avoidance, and autonomously returns for a precision landing on a custom recharge station.

H: RL Control Integration

Future Target Reinforcement Learning PPO Adaptive Control

Transitioning the core flight navigation pipeline to utilize Reinforcement Learning. Training the agent to handle complex aerodynamic disturbances and edge-case obstacle avoidance directly from sensory input, hardening the system for unpredictable real-world dynamics prior to physical deployment.

I: Semantic World Modeling

Future Target Semantic Segmentation 3D Landmarks Spatial Memory

Upgrading the environmental representation from pure geometric voxels to semantically aware clusters. The drone will process raw spatial data into known, "textable" 3D landmarks, bridging the gap between physical geometry and cognitive understanding of the environment.

J: Natural Language HRI & HIL

Future Target LLM Integration HRI Human-in-the-Loop

The final RaaS deployment layer. Developing a Human-Robot Interaction (HRI) interface where operators can issue natural language commands. The system will parse these commands against the semantic world model to autonomously generate and execute complex, situational inspection tasks.
